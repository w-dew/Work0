# 实验项目：基于 Taichi 的万有引力粒子群仿真系统

## 1. 项目架构
本项目采用模块化设计，将硬件初始化、物理模拟与参数控制分离，确保了代码的可维护性。

* **main.py (Root)**: 程序的启动入口。
* **src/Work0/**: 核心代码包。
    * **config.py**: 集中管理物理常数（如引力强度、阻尼系数）与渲染配置。
    * **physics.py**: 核心逻辑层。定义 GPU Field 并实现基于 Taichi Kernel 的并行物理计算。
    * **main.py (Internal)**: 负责 GUI 生命周期的管理与实时渲染调度。

---

## 2. 代码逻辑
系统模拟了一个处于受限空间内的动态粒子群，其运动受以下物理逻辑驱动：

### 2.1 并行数据结构
利用 `ti.Vector.field` 在显存中开辟连续空间，使得数万个粒子的位置和速度可以由 GPU 并行处理，极大提升了计算效率。

### 2.2 核心算法：引力与阻尼模型
粒子的状态更新遵循牛顿运动定律的离散形式，核心代码实现如下：

```python
@ti.kernel
def update_particles(mouse_x: float, mouse_y: float):
    for i in range(NUM_PARTICLES):
        # 1. 计算方向：粒子指向鼠标位置的向量
        mouse_pos = ti.Vector([mouse_x, mouse_y])
        dir = mouse_pos - pos[i]
        
        # 2. 模拟引力：给粒子一个向心加速度
        if dir.norm() > 0.05:
            vel[i] += dir.normalized() * GRAVITY_STRENGTH
            
        # 3. 模拟阻力与位移更新 (DRAG_COEF = 0.98)
        vel[i] *= DRAG_COEF  
        pos[i] += vel[i] 
        
        # 4. 边界处理：撞墙反弹逻辑
        for j in ti.static(range(2)):
            if pos[i][j] < 0 or pos[i][j] > 1:
                # 碰到边界时速度反向并折损
                vel[i][j] *= BOUNCE_COEF
```

### 2.3 物理逻辑详细说明
* **引力计算**：通过计算粒子与鼠标的单位方向向量，结合 `GRAVITY_STRENGTH` 实时更新速度，实现粒子随鼠标移动的效果。
* **能量损耗**：通过 `DRAG_COEF` 模拟空气阻力，防止系统动能无限增加导致粒子运动失控。
* **边界约束**：当粒子坐标超出 [0, 1]范围时执行反弹逻辑，速度分量反转并乘以 `BOUNCE_COEF`，模拟真实碰撞中的能量吸收。

## 3. 实现功能
* **万级粒子实时仿真**：利用 GPU 并行计算，在普通集成显卡上驱动 **10,000+** 粒子。
* **动态鼠标交互**：粒子群实时追踪鼠标位置，形成类似“星团”的聚拢、绕转与扩散效果。
* **物理边界反馈**：完整的四周边界碰撞检测，粒子在窗口边缘具有真实的反弹物理表现。
* **参数可调性**：通过修改 `config.py`，可自由切换粒子数量、颜色及物理环境常数。
